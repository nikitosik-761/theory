compile (default): доступны в пути к классам проекта во всех задачах сборки. эти зависимости так же транзитивны.

provided: зависимостей, которые должны предоставляться во время выполнения JDK или контейнером.

test: Зависимости тестов не являются транзитивными и присутствуют только для путей к классам тестирования и выполнения.

system: как provided, но нужно указывать конкретный jar-файл в системе.

runtime: Зависимости с этой областью необходимы во время выполнения. Но для компиляции кода проекта они нам не нужны. зависимости будут не видны во время компиляции.

import: Он доступен только для типа зависимости pom. указывает на pom файл, который будет содержать зависимости.


зависимости provided, system и test никогда не будут включены в основной проект.
