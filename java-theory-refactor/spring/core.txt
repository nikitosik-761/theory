как спринг выбирает bean: 
	по типу данных
	по имени поля (соответстующий имени класса)
	по интерфейсам (не факт)

construct, setter, reflection 
construct: 
	позволяет провести иницилизацию при старте, что позволяет избежать null pointer
	setter позволять переконфигурировать bean
	reflection нежелателен, т.к уменьшает поддерживаемость, возможны ошибки и тд.
