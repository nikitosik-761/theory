read uncommited: 	
	1) грязное чтение: чтение незакомиченных данных, которые потом откатятся.

read commited:
	2) потерянные изменения (грязная запись): когда несколько транзакций изменяют одни и те же данные, последняя транцакция затирает изменения всех предыдущих. (если используется неатомарная операция, то есть данные сначала считываются в переменную, изменяются и записываются обратно в бд)

	3) неповторяющееся чтение: транзакция при чтении одних и тех же данных получает разные значения, потому что другие транзакции меняют данные.

repeatable read: 
	4) чтение фантомов: транзакция при одной и той же выборке данных получает разные набор строк, потому что другое транзакции добавляет новые строки.
